# Security Policy

## Reporting a vulnerability

ArgorixLang is a security- and cryptography-focused project, so we take
vulnerability reports seriously.

**Please do not report security issues through public GitHub issues, discussions,
or pull requests.**

Instead, report privately using
[GitHub Security Advisories](https://github.com/argorixlabs/argorixlang/security/advisories/new).
If you cannot use that, email the maintainers at **security@argorixlabs.dev**.

Please include:

- A description of the vulnerability and its impact.
- Steps to reproduce (a minimal `.argx` snippet or bytecode artifact is ideal).
- Affected version(s) or commit SHA.

## Response targets

- Acknowledgement within **3 business days**.
- An initial assessment and remediation plan within **10 business days**.
- We will coordinate a disclosure timeline with you and credit you unless you
  prefer to remain anonymous.

## Supported versions

The project is pre-1.0; only the latest released minor version receives security
fixes.
