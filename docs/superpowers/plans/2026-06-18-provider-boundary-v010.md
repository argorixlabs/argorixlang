# Argorix Lang v0.10 Provider Boundary Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a synchronous, simulated-only provider boundary for all reactive tool and model calls.

**Architecture:** A standalone `argorix_provider` crate defines provider contracts and a default registry. Compilation preserves optional tool providers in the AST, validates them semantically, resolves defaults in IR, and emits explicit Bytecode 0.10. The VM owns the registry, invokes it after authorization, validates responses fail-closed, and exposes audit events and summaries.

**Tech Stack:** Rust 2021, Cargo workspace, serde, thiserror, clap.
